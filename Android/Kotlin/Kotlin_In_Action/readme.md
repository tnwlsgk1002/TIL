## 1장 코틀린이란 무엇이며, 왜 필요한가?
---
### 1.1 코틀린 대상 플랫폼
---
- 자바가 실행되는 모든 곳
- 서버상의 코드, 안드로이드 디바이스에서 실행되는 모바일 애플리케이션
- 그 외에도 IOS 디바이스, 데스크탑 애플리케이션에도 실행 가능

### 1.2 코틀린 특성
---
1. 정적 타입(statically typed) 지정 언어 : 성능, 신뢰성, 유지 보수성, 도구 지원
2. 객체지향 프로그래밍 + 함수형 프로그래밍
3. 무료 오픈소스

### 1.3 코틀린 응용
---
1. 코틀린 서버 프로그래밍
2. 코틀린 안드로이드 프로그래밍

### 1.4 코틀린 철학
---
1. 실용성
2. 간결성
3. 안전성

## 2장 코틀린 기초
---

### 2.1 기본 요소: 함수와 변수
---
- 함수를 최상위 수준에 정의할 수 있다. 자바와 달리 꼭 클래스 안에 함수를 넣어야 할 필요가 없다.
- 배열도 일반적인 클래스와 마찬가지다. 자바와 달리 배열 처리를 위한 문법이 따로 존재하지 않는다.

#### 2.1.1 함수
---
```kotlin
fun max(a: Int, b: Int) : Int {
  return if (a > b) a else b
}
```
- 함수는 `fun`으로 시작하며 뒤에 함수 이름이 온다.
- 함수 이름 뒤에는 괄호 안에 파라미터 목록이 온다.
- 함수의 반환 타입이 그 뒤에 오는데, 괄호와 반환 타입 
- 사이를 콜론(:)으로 구분한다.
- 위 코드에서 if문은 문장이 아닌 결과를 만드는 식이다. 

##### 문(statement)과 식(expression)의 구분
---
- 식은 값을 만들어내며 다른 식의 하위 요소로 계산해 참여할 수 있다.
- 반면 문은 자신을 둘러싸고 있는 가장 안쪽 블록의 최상위 요소로 존재하며 아무런 값을 만들어내지 않는다.
- 자바에서는 모든 제어구조가 문인 반면 코틀린에서는 루프를 제외한 대부분의 제어 구조가 식이다.
- 반면 대입문은 자바에서는 식이었으나 코틀린에서는 문이 됐다. 그로 인해 자바와 달리 대입식과 비교식을 잘못 바꿔 써서 버그가 생기는 경우가 없다.

##### 식이 본문인 함수
---
```kotlin
fun max(a: Int, b: Int): Int = if (a > b) a else b
```

- 중괄호를 없애고 return을 제거하면서 등호를 식 앞에 붙이면 더 간결하게 함수를 표현할 수 있다.
- 반환 타입을 생략하면 더 간결하게 만들 수 있다.

```kotlin
fun max(a: Int, b: Int): Int = if (a > b) a else b
```

- 코틀린은 정적 타입 지정 언어이므로 컴파일 시점에 모든 식의 타입을 지정해야하지 않는가?
- 식이 본문인 함수의 경우 굳이 사용자가 반환 타입을 적지 않아도 컴파일러가 함수 본문 식을 분석해서 식의 결과 타입을 함수 반환 타입으로 정해준다. = **타입 추론**
- 식이 본문인 함수의 반환 타입만 생략 가능

#### 2.1.2 변수
---

```kotlin
val question = "삶, 우주, 그리고 모든 것에 대한 궁극적인 질문"
val answer = 42 // 생략 가능
val answer: Int = 42 
```

- 타입으로 변수 선언을 시작하면 타입을 생략할 경우 식과 변수 선언을 구별할 수 없다.
- 코틀린에서는 키워드로 변수 선언을 시작하는 대신 변수 이름 뒤에 타입을 명시하거나 생략하게 허용한다. 

```kotlin
val yearsToCompute = 7.5e6
val answer : Int
answer = 42
```

- 초기화 식을 사용하지 않고 변수를 선언하려면 변수 타입을 반드시 명시해야 한다.

##### 변경 가능한 변수와 변경 불가능한 변수
---
> **val(값을 뜻하는 value에서 따옴)** : 변경 불가능한 참조를 저장하는 변수, 초기화 하고나면 재대입 불가능

> **var(변수를 뜻하는 variable에서 따옴)** : 변경 가능한 참조, 변수의 값은 바뀔 수 있음

- 기본적으로는 val, 나중에 꼭 필요할 때만 var 사용
- val 변수는 블록을 실행할 때 한 번만 초기화
	-  but, 어떤 블록이 실행 될 때 오직 한 초기화 문장만 실행됨을 컴파일러가 확인 => 여러 값으로 초기화 가능

```kotlin
val languages = arrayListOf("Java") // 불변 참조 선언
languages.add("Kotlin") // 참조가 가리키는 객체 내부를 변경
```

- val 참조 자체는 불변이나 참고가 가리키는 객체의 내부 값은 변경 가능하다.

```kotlin
var answer = 42
answer = "no answer" // Error: type mismatch
```

- 그러나 var 변수라도 변수의 타입은 변경할 수 없다.
- 어떤 타입의 변수에 다른 타입의 값을 저장하고 싶다면 변환 함수를 써서 값을 변수의 타입으로 반환하거나 값을 변수에 대입할 수 있는 타입으로 강제 형 변환한다.

#### 2.1.3 더 쉽게 문자열 형식 지정: 문자열 템플릿
---
```kotlin
fun main(args: Array<String>) {
	val name = if (args.size > 0) args[0] else "Kotlin"
	println("Hello, $name!")
}
```

- 문자열 리터럴에 필요한 곳에 변수를 넣고 앞에 $ 추가
- $ 문자를 넣고 싶다면 `println("\$x")` 와 같이 \를 사용해 $을 이스케이프 시켜야 한다.

```Kotlin
fun main(args: Array<String>) {
	if (args.size > 0) {
		println("Hello, $args{0}!") // ${} 사용
	}
}
```

- 복잡한 식은 중괄호({})로 둘러싸서 문자열 템플릿 안에 넣으면 된다.

##### 한글을 문자열 템플릿에서 사용할 경우 주의할 점
---
```kotlin
var name = "sujin"
println("$name님 반가워요") // unresolve reference
println("${name}님 반가워요") // 올바로 동작
```

- 코틀린 또한 변수 이름에 한글이 들어갈 수 있다.
- 문자열 템플릿 안에 $로 변수를 지정할 때 변수명 바로 뒤에 한글을 붙여서 사용하면 코틀린 컴파일러는 영문자와 한글을 한꺼번에 식별자로 인식해서 unresolve reference 오류를 발생시킨다.
- 이 문자를 해결하기 위해 ${}을 사용한다.

### 2.2 클래스와 프로퍼티
---
- 자바에서는 생성자 본문에 반복적으로 코드가 들어가는 경우가 많다.
- 코틀린에서는 이런 필드 대입 로직을 훨씬 더 적은 코드로 작성할 수 있다.

#### 2.2.1 프로퍼티
---
- 클래스 : 데이터를 캡슐화하고 캡슐화한 데이터를 다루는 코드를 한 주체 아래 가두기 위해 존재
- 자바에서는 데이터 필드에 저장, 접근자 메소드 제공
- 자바에서는 필드와 접근자를 한데 묶어 프로퍼티라고 부른다.
- 코틀린에서는 프로퍼티를 언어 기본 기능으로 제공하며 자바의 필드와 접근자 메소드를 완전히 대신 한다.
- val로 선언한 프로퍼티는 읽기 전용으로 (비공개) 필드와 필드를 읽는 단순한 (공개) 게터를 만든다.
- var로 선언한 프로퍼티는 변경 가능으로 (비공개) 필드, (공개) 게터, (공개) 세터를 만든다.

```Kotlin
val name = Person("Bob", true) // new 없이 호출
println(person.name) // 프로퍼티 이름을 직접 사용해도 코틀린이 자동으로 게터 호출
println(person.isMarried)
```

- 대부분의 프로퍼티에는 프로퍼티를 저장하는 필드가 있으며 이를 프로퍼티를 **뒷받침하는 필드**라고 부른다.
